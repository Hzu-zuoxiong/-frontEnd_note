# 前端考点

* 你用过哪些前端性能优化的方法？

```
1.减少http请求次数：CSS精灵图，JS、CSS源码压缩，CDN托管，data缓存
2.用innerHTML代替DOM操作，减少DOM操作次数，优化JS性能
3.当需要设置的样式很多时，设置className而不是直接操作style
4.少用全局变量、缓存DOM节点查找的结果、减少IO读取操作
5.图片预加载，将样式表放在顶部，脚本放在底部
6.避免在页面的主题布局中使用table，table要等其中的内容完全下载之后才会显示出来，显示比div+css布局慢
```

* 什么叫优雅降级和渐进增强？

```
优雅降级：Web站点在所有新式浏览器中都能正常工作，如果用户使用的是老式浏览器，则代码会检查以确认它们是否能正常工作，由于IE独特的
         盒子模型布局问题，针对不同版本的IE的hack实践优雅降级，为那些无法支持功能的浏览器增加候选方案，使之在旧式浏览器上以某种
         形式降级体验却不至于完全失效。
渐近增强：从被所有浏览器支持的基本功能开始，逐步地添加那些只有新式浏览器才支持的功能，向页面增加无害于基础浏览器的额外样式和功能。
         当浏览器支持时，它们会自动地呈现出来并发挥作用。
```

* 对前端界面工程师这个职位是怎样理解的？它的前景会怎样？

```
前端是最贴近用户的程序员，比后端、数据库、产品经理、运营、安全都近。
1.实现界面交互
2.提升用户体验
3.有了Node.js，前端可以实现服务端的一些事情

前景：
1.前端是最贴近用户的程序员，前端的能力就是能让产品从90分进化到100分，甚至更好
2.参加项目，快速高质量完成实现效果图，精确到1px
3.与团队成员，UI设计，产品经理沟通
4.做好的页面结构，页面重构和用户体验
5.处理hack，兼容、写出优美的代码格式
6.针对服务器的优化、拥抱最新前端技术
```

* 平时如何管理你的项目？

```
1.前期团队必须确定好全局样式（globe.css），编码模式（utf-8）等
2.编写习惯必须一致（例如都是采用继承式的写法，单样式都写成一行）
3.标注样式编写，各模块都及时标注（标注关键调用的地方）
4.页面进行标注（例如页面模块开始和结束）
5.CSS跟HTML分文件夹并行存放，命名都统一
6.JS分文件夹存放命名以该JS功能为准的英文翻译
7.图片采用整合的images.png尽量整合一起使用方便将来的管理
```
